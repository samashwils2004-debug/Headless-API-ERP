3. TENANT ENFORCEMENT LAYER  
tenantGuard.ts  
Responsibility:  
• Block rendering without project context  
• Attach institution_id and project_id to every request  
• Scope Zustand stores  

• Reset state on project change  
Enforcement Logic  
export function assertTenantContext(projectId?: string) {  
  if (!projectId) {  
    throw new Error("Tenant context missing. Rendering blocked.");  
  } 
} 
Axios Interceptor  
api.interceptors.request.use((config) => {  
  const { institutionId, projectId } = useProjectContextStore.getState();  
 
  if (!institutionId || !projectId) {  
    throw new Error("Tenant context required.");  
  } 
 
  config.headers["X -Institution -ID"] = institutionId;  
  config.headers["X -Project -ID"] = projectId;  
 
  return config;  
}); 
    This guarantees:  
No API request can be made without tenant context.

5. VALIDATION ENFORCEMENT LAYER  
validationGuard.ts

Enforcement rule:  
export function assertDeployAllowed(validationResult) {  
  const { schema, graph, permissions, compliance } = validationResult;  
 
  if (!schema.valid ||  
      !graph.valid ||  
      !permissions.valid ||  
      !compliance.compliant) {  
    throw new Error("Deployment blocked by validation.");  
  } 
}

    Guarantees:

6. DEPLOYMENT GUARD  
deploymentGuard.ts  

Prevents:  
• Double deployment  
• Auto -deploy  
• Concurrent deployment  
let isDeploying = false;  
 
export async function guardedDeploy(fn) {  
  if (isDeploying) {  
    throw new Error("Deployment already in progress.");  
  } 
 
  isDeploying = true;  
  try { 
    return await fn();  
  } finally {  
    isDeploying = false;  
  } 
} 
    Guarantees:  
No race conditions.
